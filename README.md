# TOTP 临时密码生成器

与微信小程序"临时密码生成器"完全兼容的 TOTP 密码生成工具，纯 Python 标准库实现，无外部依赖。

## 使用方法

### 生成临时密码
```bash
python3 generate_totp.py <管理员数字密码> <有效时长(3-60分钟)>
```

示例：

```bash
python3 generate_totp.py 123456 5
```

输出为 10 位密码，前 8 位为动态密码，后 2 位为有效期分钟数。

### 验证有效期修改漏洞
```bash
python3 verify_exploit.py <管理员数字密码> <初始有效期(3-60分钟)>
```

示例：

```bash
python3 verify_exploit.py 123456 5
```

该工具可以检测是否存在通过修改密码后两位有效期数字来延长密码有效期的安全漏洞。

## TOTP 生成原理

1. 时间计数器从 2000 年 1 月 1 日开始计算，按指定有效期步长递增
2. 使用管理员密码作为密钥，时间计数器作为消息，计算 HMAC-SHA1 摘要
3. 对摘要进行动态截断生成 32 位整数
4. 转换为 8 位数字密码（低位在前，最后一位固定为 0）
5. 追加 2 位有效期，组成最终 10 位密码

## 安全漏洞说明

该TOTP实现存在安全漏洞：攻击者可以通过修改密码的后两位有效期数字，将原本短有效期的密码延长为更长的有效期，从而在未来的时间点仍然可以使用该密码。

漏洞原理：
1. 密码生成时，前8位基于时间计数器和密钥生成，后2位是有效期的明文字段
2. 不同有效期的时间计数器可能在特定时间点产生相同的计数器值
3. 攻击者可以将短有效期密码的后两位修改为更长的有效期，从而在对应的时间窗口使用

`verify_exploit.py`工具可以检测这种漏洞的存在。
